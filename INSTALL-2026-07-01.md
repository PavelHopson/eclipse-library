# Установки 30.06-01.07.2026 - план внедрения по проектам

> Снапшот: что из нового батча реально использовать, что держать как reference,
> а что не тащить в продукты. Статусы: target high-value, queue, reference,
> hold / do-not-integrate.

## target high-value

### OpenHuman -> Eclipse Chat AI Memory · Hopson Sentinel · Smart Life Assistant

Смысл не в копировании OpenHuman, а в паттерне постоянного контекста.

- **Eclipse Chat:** room memory, project memory, "since you were away", semantic search по решениям/файлам/задачам.
- **Hopson Sentinel:** локальный оператор, который помнит привычки, рабочие окружения, частые команды и контекст задач.
- **Smart Life Assistant:** персональная память целей, расписания, привычек и follow-up действий.
- Обязательное условие: consent, scoped permissions, audit trail, возможность удалить память.

### Cloudflare Is It Agent Ready -> все публичные сайты

Прогонять как чеклист для сайтов, которые должны индексироваться AI-агентами.

- **Eclipse Forge Landing:** повысить AI Search / SEO / agent-readable структуру.
- **Eclipse Library:** сделать библиотеку удобной для AI-индексации и цитирования.
- **Eclipse Chat landing:** перед публичным запуском проверить discoverability.
- **Клиентские сайты:** добавить как отдельную услугу "agent-ready audit".

### Loopy -> oh-my-claudecode · eclipse-library · agent governance

Нужен как слой порядка над skills/MCP/automations.

- Проверять дубли, пересечения, рискованные действия и сломанные loops.
- Использовать вместе с SkillSpector-подходом перед массовой установкой skills.
- Цель: не дать агентской среде превратиться в склад случайных автоматизаций.

## queue

### OmniRoute -> Eclipse AI Hub · Hopson Sentinel · Eclipse Chat agents

Референс для provider-router: fallback, compression, model routing.

- В продуктах использовать только легальные ключи и собственные квоты.
- Не строить бизнес на "обходе лимитов" или непонятном бесплатном доступе.
- Сначала POC: один OpenAI-compatible endpoint, 2-3 провайдера, latency/cost/error metrics.

### Google image/video low-cost tier -> Shotforge · Text2Image · Eclipse Media

Проверить как дешёвый слой генерации ассетов.

- Тест-набор: портрет, product shot, UI mock, проектная карточка, короткое видео.
- Проверить watermark, права использования, качество русского текста, цену на 100 генераций.
- Если окупается - добавить как provider в Shotforge/Text2Image.

### TREK -> Smart Life Assistant · Eclipse DnD Forge · Eclipse Chat

Ценность - UX-паттерн "workspace вокруг процесса".

- **Smart Life Assistant:** travel/life planning board: маршруты, бюджет, задачи, заметки.
- **DnD Forge:** route planning, party journal, campaign itinerary, poll механика.
- **Eclipse Chat:** client/project rooms с планом, опросами, shared checklist и budget table.

### OpenClaw Mobile -> Hopson Sentinel · Eclipse Chat operator mode

Референс мобильного пульта для AI-оператора.

- User approval flow: агент предлагает действие -> пользователь подтверждает с телефона.
- Permissions: камера, геолокация, голос, экран - только явно и временно.
- Не интегрировать чужое приложение; использовать как UX benchmark.

- Сделать внутренний трек "LLM Engineer path".
- Вытащить темы для Educator-AI: embeddings, fine-tuning, quantization, evals, deployment.

### SimpleX Chat -> Eclipse Chat privacy model

Уже в библиотеке. Новая конкретика:

- ephemeral rooms;
- одноразовые invite links / QR;
- минимизация метаданных;
- локальное хранение чувствительных данных;
- private/client channel без лишних идентификаторов.

## hold / do-not-integrate

### Torlink

Только как reference для CLI UX, очередей загрузки и проверки источников.

- Не делать публичный downloader "любых файлов".
- Не использовать на copyrighted content.
- Тестировать только open-data/public-domain файлы в sandbox.

### Aliens Eye

Уже в библиотеке как self-audit цифрового следа.

- Не интегрировать в продукты.
- Не использовать для "пробива" людей.
- Возможный легитимный сценарий: проверка собственного/клиентского публичного следа с согласием.

### Seed-Audio 1.0

Уже в библиотеке как audio/TTS reference.

- Voice cloning только со своим голосом или явным согласием.
- Для продукта нужен consent gate и журнал генераций.

### Uncensored Qwen / "Claude Opus" HF model

Не интегрировать.

- Сомнительный провенанс и high-risk uncensored позиционирование.
- Нельзя использовать в Eclipse Chat agents, Sentinel по умолчанию, клиентских задачах или production.
- Максимум isolated red-team / safety-eval без доступа к реальным данным.
