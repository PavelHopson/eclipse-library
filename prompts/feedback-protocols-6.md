# Feedback Protocols — 6 промптов для управления диалогом с ИИ

> Источник: Telegram «Не баг, а фича» 04.06.2026 (компиляция с Reddit).
> Не про «что спросить», а про то, **как заставить ИИ вести диалог
> по-человечески** — собирать контекст, не угадывать, тормозить когда
> убегает вперёд. Короткие протоколы, которые вставляются в любой момент сессии.
>
> **Куда применяем:** рабочий стиль с Claude Code / Eclipse Hopson Sentinel /
> Eclipse AI Hub — особенно `#3` и `#6` ловят главную проблему длинных
> агентных сессий: агент «угадал» направление и понёсся не туда.
> Дополняет [skeptical-verifier](skeptical-verifier.md) (критичность ответа)
> и [Anti-AI-Text](anti-ai-text-6-prompts.md) (стиль вывода).

---

## 1️⃣ Сбор информации перед ответом
```
Задавай вопросы, пока не соберёшь достаточно информации для обратной связи.
По возможности предлагай варианты ответов вместе с вопросами. Задавай вопросы
по одному.
```

## 2️⃣ Уточнение формулировки (расширение контекста)
```
Уточни эту формулировку, добавив подробности, нюансы, ясность и конкретику,
сохранив первоначальный замысел. Усовершенствуй логику, не изменяя сути запроса.
```

## 3️⃣ Открытый диалог (анти-«угадайка»)
```
Не делай предположений. Не скрывай затруднений. Открыто обсуждай компромиссы.
Вноси минимальные изменения, необходимые для решения проблемы. Вмешивайся
только в то, что необходимо. Убирай за собой. Определи критерии успеха.
Повторяй цикл, пока результаты не будут подтверждены.
```

## 4️⃣ Лаконичность вместо болтовни
```
Будь лаконичным, критичным и информативным, а не болтливым и дружелюбным;
избегай клише. Задавай уточняющие вопросы. Сузь круг вопросов и давай
конкретные практические советы.
```

## 5️⃣ Протокол «Дум-Дум» (переосмысление тупика)
```
Протокол «Дум-Дум»: если мы ломаем голову над проблемой и не можем её решить —
останавливаемся и переосмысляем, действительно ли это лучший способ.
```

## 6️⃣ Протокол «Подожди, блин» (синхронизация темпа)
```
Протокол «Подожди, блин»: иногда ИИ опережает меня в рекомендациях. При
активации этого протокола мы останавливаемся и убеждаемся, что оба на одной волне.
```

---

**Паттерн использования:** `#1`+`#3` в начале новой задачи (агент собирает
требования, а не угадывает) → `#5`/`#6` как стоп-краны в середине, когда
сессия уходит в сторону. `#3` особенно ценен для `/loop` и долгих
оркестрационных прогонов — формализует «определи критерии успеха, повторяй до подтверждения».
