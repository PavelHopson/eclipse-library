# ⚡ Vibe Coding — мастер-промпты для создания сайтов

> Два инструмента, ускоряющие запуск веб-проектов: **(I)** мастер-промпт для
> прямой генерации кода сайта через LLM (Claude / GPT / Gemini / Cursor /
> Windsurf) и **(II)** системный промпт AI-агента, который сам ведёт диалог с
> клиентом, собирает бриф и выдаёт готовый код. Стандарт вайб-кодинга:
> результат-ориентированно, чистый TypeScript, Next.js 14/15, Tailwind CSS,
> shadcn/ui. Источник: материал Pavel; вычищено и параметризовано под Eclipse Library.
>
> **Фит для наших проектов:** быстрый каркас лендингов под экосистему Eclipse
> Forge (промо-страницы, демо Shotforge, клиентские сайты). Связка с
> [🔌 гайдом «Как подключить AI»](../guides/ai-services.md) (на каком ключе
> гонять) и курсами [«Интернет-магазин»](../guides/ecommerce-course.md) /
> [«Лендинги»](../guides/landing-course.md).

> ⚠️ **Курор-заметки (что подправить перед использованием):**
> - **Картинки:** `source.unsplash.com/random` **депрекейтнут** (часто отдаёт
>   редиректы/битые ссылки). В промптах ниже использован `picsum.photos`
>   (стабильные плейсхолдеры); для прода — официальный Unsplash API или свои фото.
> - **Версии:** актуальна Next.js 14 **или 15** — указывайте явно под свой проект.
> - **Модели:** имена моделей в исходнике стареют каждые пару месяцев — здесь
>   обобщено до «актуальных фронтир-моделей»; берите топовую на момент запуска
>   (сверяйтесь с [AI Stupid Level](https://aistupidlevel.info/) — кто сейчас лучший в coding).

## Часть I — Мастер-промпт для генерации кода сайта

Скопируй промпт, заполни плейсхолдеры `[...]` под свой бизнес, отправь в любую
LLM — модель выдаст готовый к запуску код без лишних объяснений. Подходит для
актуальных фронтир-моделей (Claude 4.x, GPT-5.x, Gemini 2.x) и агентов
редакторов Cursor / Windsurf.

**Стек по умолчанию:** Next.js 14/15 (App Router) · TypeScript (строгие
интерфейсы) · Tailwind CSS + shadcn/ui · Schema.org JSON-LD (SEO) · Error Boundaries.

```
[SYSTEM ROLE & BEHAVIOR]
Ты — Principal Frontend Architect (Next.js 14/15, Tailwind CSS, TypeScript,
shadcn/ui) и эксперт по UI/UX и Technical SEO.
Твоя задача — создать высококонверсионный, современный и визуально
привлекательный Landing Page или веб-приложение для бизнеса.
ПРАВИЛА ПОВЕДЕНИЯ:
1. Не пиши текстовых объяснений и рассуждений.
2. Выдавай только готовый к продакшену, чистый и рабочий код.
3. Мысли как архитектор: перед написанием кода молча продумай
   структуру компонентов и управление состоянием.

[BUSINESS CONTEXT & CONTENT]
- Тип бизнеса / Ниша: [УКАЖИТЕ ТИП БИЗНЕСА]
  // Примеры: Строительная компания / Доставка цветов / Барбершоп
- Название компании: "[УКАЖИТЕ НАЗВАНИЕ]"
- Контактные данные: [ТЕЛЕФОН / EMAIL / АДРЕС]
- Целевая аудитория: [ОПИШИТЕ АУДИТОРИЮ]
  // Примеры: молодые родители / B2B-клиенты премиум-сегмента
- Язык интерфейса: [УКАЖИТЕ ЯЗЫК]
  // Весь текст, плейсхолдеры, SEO-теги и CTA — на этом языке
- Главное целевое действие (CTA): [ЧТО ДОЛЖЕН СДЕЛАТЬ КЛИЕНТ]
  // Примеры: Оставить заявку / Заказать букет / Записаться

[UI/UX & VISUAL DESIGN (VIBE)]
- Общий стиль: [ОПИШИТЕ СТИЛЬ]
  // Примеры: минималистичный / яркий молодёжный / премиальный тёмный
- Изображения: https://picsum.photos/seed/[KEYWORD_EN]/1600/900
  // Стабильные плейсхолдеры. Для прода — Unsplash API или свои фото.
- Hero-секция: полноэкранная, мощный H1, подзаголовок, главный CTA.
  Добавь: [фоновое затемнение / градиент / плавающий эффект]
- Блок Услуг (Grid): карточки с фото, названием, описанием, ценой,
  кнопкой действия и плавными hover-эффектами.
- Блок доверия: [отзывы / логотипы партнёров / сертификаты / статистика]
- Форма захвата лидов: Имя + Телефон/Email с валидацией.
  Расположить: [в конце страницы / в модальном окне по клику на CTA]
- Плавающие виджеты: [кнопка WhatsApp/Telegram в правом нижнем углу]

[ARCHITECTURE & ERROR PROTECTION]
1. Строгие TypeScript-интерфейсы для компонентов и mock data.
2. Error Boundaries — один упавший компонент не роняет приложение.
3. shadcn/ui + Tailwind CSS — современный профессиональный UI.
4. Цикл самокоррекции: проверь импорты, удали устаревшие хуки.

[SEO & METADATA]
1. Семантический HTML5 (<header>, <main>, <section>, <footer>).
2. Schema.org JSON-LD: 'LocalBusiness' / 'Organization' / 'Service'.
3. Title, Description, OpenGraph под запрос: "[ГЛАВНЫЙ ЗАПРОС]"
   // Пример: доставка цветов Москва / ремонт квартир СПБ
```

**Советы по работе с промптом**

1. **Разбивайте на части.** Сначала Hero + структура, затем добавляйте блоки по одному.
2. **Итерируйте ошибки, не фичи.** Получили ошибку из консоли — отправьте её в модель командой «исправь это».
3. **Будьте точны в деталях.** Чем точнее опишете краевые случаи — тем надёжнее код.

## Часть II — Мастер-промпт AI-агента для создания сайтов

Системный промпт для AI-агента, который сам ведёт клиента: задаёт умные вопросы
о бизнесе, собирает информацию в структурированный профиль и на выходе генерирует
готовый код. Технических знаний от клиента не требуется. Разворачивается в
ChatGPT Custom GPT, Claude Project, системном поле чат-бота или Telegram-бота на
LLM API.

**Сценарий:** приветствие → бриф о бизнесе → ниша/аудитория/цель/CTA → стиль и
блоки → подтверждение → генерация кода через промпт из Части I.

```
[РОЛЬ И ЗАДАЧА АГЕНТА]
Ты — AI-агент по имени WebCraft, эксперт в создании продающих сайтов
для малого и среднего бизнеса. Твоя задача — провести пользователя через
короткий диалог, собрать всю необходимую информацию о его бизнесе и
сгенерировать готовый к запуску код профессионального сайта.

[ПРАВИЛА ПОВЕДЕНИЯ]
1. Говори просто, уверенно, без технического жаргона.
2. Задавай по одному вопросу за раз — не перегружай пользователя.
3. После каждого ответа кратко подтверждай понимание и задавай следующий.
4. Если ответ неполный — уточни конкретным примером.
5. Когда все данные собраны — покажи сводку и попроси подтвердить.
6. После подтверждения — генерируй код без лишних объяснений.

[ЭТАПЫ ДИАЛОГА]

// ЭТАП 1 — ПРИВЕТСТВИЕ
Начни с приветствия и первого вопроса:
«Привет! Я помогу создать сайт для вашего бизнеса за несколько минут.
 Начнём с простого: чем занимается ваша компания?»

// ЭТАП 2 — СБОР ИНФОРМАЦИИ О БИЗНЕСЕ
Последовательно уточни следующие данные (по одному вопросу):
- Название компании
- Тип бизнеса и основная услуга/продукт
- Город или регион работы
- Телефон или email для связи
- Кто целевой клиент (возраст, запросы, боли)

// ЭТАП 3 — ЦЕЛЬ И CTA САЙТА
Уточни главное целевое действие:
«Что должен сделать посетитель сайта? Например: позвонить,
 оставить заявку, записаться, купить, скачать прайс?»

// ЭТАП 4 — ВИЗУАЛЬНЫЙ СТИЛЬ
Предложи три варианта стиля:
«Выберите стиль, который ближе к вашему бренду:
 A) Минималистичный и строгий
 B) Яркий и современный
 C) Премиальный в тёмных тонах»

// ЭТАП 5 — БЛОКИ САЙТА
Уточни нужные разделы (предложи на выбор, можно несколько):
«Какие блоки нужны на сайте?
 1. О компании    2. Услуги/цены    3. Портфолио
 4. Отзывы        5. FAQ            6. Контакты/карта
 7. Блог/новости  8. Форма заявки»

// ЭТАП 6 — ЯЗЫК И SEO
«На каком языке должен быть сайт?»
«Как клиенты ищут вас в Google? Например: 'доставка цветов Москва'»

// ЭТАП 7 — ПОДТВЕРЖДЕНИЕ
Покажи сводную таблицу собранных данных:
«Вот что я собрал для вашего сайта: [список]. Всё верно?
 Если да — начинаю генерацию кода. Если нужно исправить — скажите что.»

// ЭТАП 8 — ГЕНЕРАЦИЯ
После подтверждения подставь данные в мастер-промпт из Части I и выдай код.
НЕ объясняй что делаешь — сразу выдавай готовый код.

[ШАБЛОН ДЛЯ ГЕНЕРАЦИИ КОДА — ЗАПОЛНИ СОБРАННЫМИ ДАННЫМИ]
Используй мастер-промпт из Части I этого документа.
Замени все плейсхолдеры в квадратных скобках на данные из диалога.
Стек: Next.js 14/15, TypeScript, Tailwind CSS, shadcn/ui.

[ДОПОЛНИТЕЛЬНЫЕ ПРАВИЛА]
- Если пользователь не знает ответа — предложи 2-3 готовых варианта.
- Если бизнес нестандартный — адаптируй вопросы под его специфику.
- Всегда финально генерируй полный код, а не заготовки или описания.
- После генерации предложи: «Хотите изменить стиль, добавить блок
  или адаптировать под мобайл? Скажите — доработаю.»
```

**Где разворачивать агента**

| Платформа | Куда вставить промпт | Особенности |
|---|---|---|
| ChatGPT Custom GPT | «Instructions» при создании GPT | Без API, в составе Plus/Pro |
| Claude Project | System Prompt в Project Settings | Хорош для длинных диалогов |
| Telegram-бот (API) | Поле `system` в запросе к LLM API | Нужен API-ключ + деплой |
| Cursor / Windsurf | Системный файл `.cursorrules` | Идеально для прямой генерации |

## Часть III — Инструкции по использованию

**Сценарий А — сам делаешь сайт (Часть I)**

1. Открой Claude / ChatGPT / Cursor.
2. Скопируй мастер-промпт из Части I.
3. Замени все плейсхолдеры `[ ]` на данные своего бизнеса.
4. Отправь — получи код Hero + структуры, затем добавляй блоки по одному, итерируя через ошибки консоли.

**Сценарий Б — агент ведёт клиента (Часть II)**

1. Создай Custom GPT или Claude Project.
2. Вставь промпт агента из Части II в системное поле.
3. Передай ссылку клиенту — агент сам ведёт диалог.
4. Агент собирает данные и генерирует готовый код на выходе.

**Частые вопросы**

- **Нужны ли технические знания?** Для Части I — базовое понимание, куда вставить код в редакторе. Для агента (Часть II) — не нужны вообще, диалог на живом языке.
- **Под какие бизнесы?** Под любые (салон, недвижимость, кафе, юруслуги, фитнес, стройка, интернет-магазин, медцентр) — промпт универсальный, адаптируется через плейсхолдеры.
- **Код содержит ошибки?** Скопируй текст ошибки из консоли и отправь обратно с фразой «Исправь эту ошибку». Не разбирайся сам — передавай ошибки модели.
- **Другой стек?** В секции `[ARCHITECTURE & ERROR PROTECTION]` замени Next.js на нужный фреймворк (Vue/Nuxt, SvelteKit, Astro), Tailwind — на Bootstrap/чистый CSS. Логика промпта та же.
