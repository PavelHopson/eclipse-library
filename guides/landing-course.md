# 🎓 Курс: лендинги с нуля — учебный формат

> Мини-курс для команды: научиться делать **продающие лендинги** быстро и качественно,
> бесплатно на старте, **собирая их нейросетью** (Claude / Codex / Kimi) и нашими сервисами.
> Каждый урок устроен одинаково:
> **🎯 чему научишься · 📖 коротко (теория) · 🛠 практика (с нейросетью) · ✅ чек.**
> Инструменты по этапам → [раздел библиотеки](../README.md#-лендинги--учебный-курс) · сайт: <https://library.eclipse-forge.ru>

---

## Программа курса

| Модуль | О чём | Уровень |
|---|---|---|
| **0. Что такое лендинг** | Анатомия, зачем, из чего состоит | новичок |
| **1. Первый лендинг за час** | Стек + сборка из шаблона нейросетью | новичок |
| **2. Контент, который продаёт** | Оффер, структура, визуал | базовый |
| **3. Дизайн «дорого»** | Типографика, UI-киты, анимации | базовый |
| **4. Конверсия и аналитика** | Формы, SEO, скорость, метрики | средний |
| **5. Запуск** | Хостинг, домен, чеклист | средний |
| **6. Дальше** | Переиспользование, A/B, дизайн-система | продвинутый |
| **🏁 Финальный проект** | Реальный лендинг под клиента | — |

**Как проходить:** по порядку, делая практику в каждом уроке. К концу — рабочий лендинг и навык собирать новые за часы.

## Модуль 0. Что такое лендинг

### Урок 0.1 — Анатомия лендинга
**🎯 Научишься:** видеть лендинг как набор стандартных секций и понимать, зачем каждая.
**📖 Коротко:** лендинг (landing page) — одностраничный сайт под **одно действие** (заявка, покупка, подписка). Типовые секции сверху вниз:

| Секция | Зачем |
|---|---|
| **Hero** (первый экран) | Захватить внимание: оффер + подзаголовок + кнопка (CTA) + визуал |
| **Выгоды / для кого** | Что человек получит, какие боли решает |
| **Как работает / процесс** | Снять непонимание шагами 1-2-3 |
| **Соцдоказательство** | Отзывы, логотипы, цифры — доверие |
| **Оффер / тарифы** | Конкретное предложение и цена |
| **FAQ** | Закрыть возражения |
| **CTA + форма** | Финальный призыв и форма захвата |
| **Футер** | Контакты, ссылки, юр-инфо |

**🛠 Практика:** открой 3 лендинга, которые тебе нравятся, и выпиши, какие из этих секций есть и в каком порядке.
**✅ Чек:** можешь назвать 6+ секций и зачем каждая.

## Модуль 1. Первый лендинг за час (AI-first)

### Урок 1.1 — Выбор стека
**🎯 Научишься:** выбирать инструмент под лендинг и не писать с нуля.
**📖 Коротко:** для лендингов лучший выбор — **Astro** (заточен под быстрые маркетинг-сайты) или **Next.js**, оба с **Tailwind CSS**. Наш стек — React/Vite/TS, так что Next.js/Astro + Tailwind родные. **Не пиши с нуля** — бери готовый шаблон и дорабатывай нейросетью.
**🛠 Практика:** реши, на чём делаешь (по умолчанию — **Astro + Tailwind** для статичного лендинга, **Next.js** если нужны формы/динамика).
**✅ Чек:** выбран стек и понятно, почему.

### Урок 1.2 — Собери лендинг нейросетью из шаблона
**🎯 Научишься:** собирать рабочий лендинг, объясняя нейросети словами.
**📖 Коротко:** берёшь AI-агента (**Claude Code / Codex / Kimi**), даёшь ему стартер и ведёшь задачами. Программировать не нужно — формулируй и проверяй.
**🛠 Практика:** скопируй нейросети:
> Создай лендинг на Astro + Tailwind. Сделай секции: hero (заголовок, подзаголовок, кнопка), выгоды (3 штуки), как работает (3 шага), отзывы, FAQ, CTA с формой, футер. Тёмная тема, аккуратная типографика. Запусти локально и дай ссылку.

Дальше уточняй: «поменяй заголовок на …», «сделай hero на весь экран», «добавь логотип».
**✅ Чек:** лендинг открывается локально, видны все секции.

### Урок 1.3 — Посмотри и поправь
**🎯 Научишься:** читать результат глазами заказчика и давать точные правки.
**📖 Коротко:** смотри как посетитель: понятен ли оффер за 5 секунд? Видна ли кнопка? Удобно на телефоне?
**🛠 Практика:** дай нейросети 3 правки словами (например: «увеличь кнопку», «сделай заголовок короче и яснее», «проверь мобильную вёрстку»).
**✅ Чек:** оффер понятен за 5 сек, кнопка заметна, на телефоне не разъезжается.

## Модуль 2. Контент, который продаёт

### Урок 2.1 — Оффер и заголовок
**🎯 Научишься:** писать цепляющий оффер по формулам.
**📖 Коротко:** заголовок = **что + для кого + выгода**. Формулы структуры текста: **AIDA** (внимание→интерес→желание→действие) и **PAS** (боль→усиление→решение). Заголовок должен обещать конкретный результат, а не «мы лучшие».
**🛠 Практика:** напиши оффер в [Eclipse AI Hub](https://hub.eclipse-forge.ru) (Copywriter) или попроси нейросеть:
> Дай 5 вариантов заголовка для лендинга [продукт] для [аудитория]. Формула: результат + для кого. Без воды и клише.

**✅ Чек:** есть заголовок, который за секунду объясняет суть и выгоду.

### Урок 2.2 — Структура и порядок секций
**🎯 Научишься:** выстраивать секции так, чтобы вести к заявке.
**📖 Коротко:** порядок ведёт по AIDA: hero (внимание) → выгоды/для кого (интерес) → как работает + соцдоказательство (желание) → оффер + CTA (действие). Одна страница — одно целевое действие.
**🛠 Практика:** составь список своих секций в нужном порядке и попроси нейросеть собрать их.
**✅ Чек:** секции идут логично к одной кнопке-действию.

### Урок 2.3 — Визуал
**🎯 Научишься:** делать картинки/иллюстрации без дизайнера.
**📖 Коротко:** визуал = доверие. Фото товара/продукта, иллюстрации, иконки, баннер для hero.
**🛠 Практика (наши сервисы):** фото/предметка → [Shotforge](https://shotforge.eclipse-forge.ru); баннеры/иллюстрации → [Text2Image](https://text2image.eclipse-forge.ru); иконки → Lucide/Phosphor (попроси нейросеть подключить); бесплатные фото → Unsplash/Pexels.
**✅ Чек:** в hero есть качественный визуал, иконки единого стиля.

## Модуль 3. Дизайн «дорого»

### Урок 3.1 — Основы премиум-вида
**🎯 Научишься:** отличать дорогой дизайн от дешёвого и просить нужное.
**📖 Коротко:** «дорого» = характерный шрифт (не дефолтный), крупная типографика с контрастом, много воздуха (отступы), один акцентный цвет, мягкие тени в тон фону, аккуратная сетка. Избегай: AI-фиолетовый градиент по умолчанию, три одинаковые колонки, нулевые отступы.
**🛠 Практика:** попроси нейросеть:
> Сделай типографику выразительной: крупный заголовок, шрифт с характером (например Unbounded/Manrope), больше воздуха между секциями, один акцентный цвет, мягкие тени.

**✅ Чек:** макет не выглядит «шаблонным», есть воздух и иерархия.

### Урок 3.2 — UI-киты и готовые секции
**🎯 Научишься:** собирать из готовых качественных блоков, а не рисовать с нуля.
**📖 Коротко:** бери готовые компоненты: **shadcn/ui** (Radix+Tailwind), **Flowbite**, **Preline** — там есть готовые hero/выгоды/тарифы/футер.
**🛠 Практика:** «используй компоненты shadcn/ui для карточек и формы», «возьми готовую секцию pricing из Flowbite».
**✅ Чек:** компоненты единообразны и аккуратны.

### Урок 3.3 — Анимации (premium-ощущение)
**🎯 Научишься:** добавлять движение, которое делает лендинг живым и дорогим.
**📖 Коротко:** появление секций при скролле (scroll-reveal), плавный hover, микро-интеракции. Инструменты: **framer-motion** (React), **GSAP**, **lottie** (готовые анимации из промпта — есть [skill](https://github.com/diffusionstudio/lottie)). Не переборщи — анимация помогает, а не отвлекает.
**🛠 Практика:** «добавь плавное появление секций при скролле через framer-motion», «вставь lottie-анимацию в hero».
**✅ Чек:** секции мягко появляются, hover-эффекты есть, не мельтешит.

## Модуль 4. Конверсия и аналитика

### Урок 4.1 — Формы заявок
**🎯 Научишься:** делать так, чтобы заявки реально доходили до тебя.
**📖 Коротко:** статичному лендингу нужен приёмник форм. Бесплатно: **Formspree**, **Web3Forms**, **Getform**, или отправка в **Telegram-бот** / Google-таблицу. Без бэкенда — это сервис принимает submit и шлёт тебе.
**🛠 Практика:** «подключи форму заявки через Web3Forms, заявки шли мне на почту [email]» (или в Telegram).
**✅ Чек:** отправил тестовую заявку — она пришла.

### Урок 4.2 — SEO, Open Graph и скорость
**🎯 Научишься:** чтобы лендинг находили и красиво репостили.
**📖 Коротко:** title + description, **Open Graph** (картинка превью в соцсетях/мессенджерах), фавикон, sitemap, schema.org. Скорость — проверка в **Lighthouse** (встроен в Chrome DevTools), цель 90+. Картинки в WebP, ленивая загрузка.
**🛠 Практика:** «добавь meta title/description и OG-теги с превью», «прогони Lighthouse и почини, что красное».
**✅ Чек:** в Lighthouse зелёные показатели, в мессенджере ссылка показывает превью.

### Урок 4.3 — Аналитика и цели
**🎯 Научишься:** видеть, что люди делают на лендинге.
**📖 Коротко:** **Яндекс.Метрика** (РФ) / **GA4** — счётчик + цели на отправку формы/клик по кнопке. Тепловые карты в Метрике показывают, докуда скроллят.
**🛠 Практика:** «подключи Яндекс.Метрику [номер], добавь цель на отправку формы».
**✅ Чек:** заходы и цель видны в кабинете аналитики.

## Модуль 5. Запуск

### Урок 5.1 — Хостинг и домен
**🎯 Научишься:** выкладывать лендинг в интернет бесплатно.
**📖 Коротко:** статичный лендинг → **Vercel** / **Cloudflare Pages** / **Netlify** (бесплатно, превью-деплои) или **наш VPS** через Caddy. Домен — поддомен `*.eclipse-forge.ru` или свой за ~150 ₽/год.
**🛠 Практика:** «задеплой на Vercel и дай ссылку» — или — «задеплой на наш VPS, домен `проект.eclipse-forge.ru`».
**✅ Чек:** лендинг открыт по рабочей ссылке на HTTPS.

### Урок 5.2 — Чеклист запуска + грабли
**🎯 Научишься:** не забыть важное перед показом клиенту.
**Чеклист:**
- [ ] Оффер понятен за 5 секунд, одна главная кнопка
- [ ] Форма работает, заявка доходит
- [ ] Мобильная вёрстка ровная (проверь на телефоне)
- [ ] Lighthouse 90+, картинки в WebP
- [ ] SEO-теги + OG-превью + фавикон
- [ ] Аналитика и цель подключены
- [ ] Юр-страницы (политика, контакты), если собираешь данные
- [ ] HTTPS, домен, редирект www

**Грабли:** длинный непонятный заголовок · кнопка теряется · тяжёлые картинки (тормозит) · форма «в никуда» · ломается на мобиле · анимаций слишком много.

## Модуль 6. Дальше (продвинутый)

- **Переиспользуемые секции** — собери свою библиотеку готовых блоков (hero, pricing, faq), чтобы новый лендинг — за час.
- **A/B-тесты** — две версии заголовка/кнопки, смотри что конвертит (метрика/PostHog).
- **Мульти-лендинг под трафик** — один шаблон, разные офферы под разные аудитории.
- **Дизайн-система** — токены (цвета/шрифты/отступы), чтобы все лендинги были в едином стиле бренда.

## 🏁 Финальный проект

Собери **реальный лендинг** (себе/клиенту/под наш проект) от начала до конца:
1. Определи цель и аудиторию → оффер (модуль 2).
2. Собери нейросетью из шаблона (модуль 1), наполни контентом и визуалом (наши сервисы).
3. Доведи дизайн (модуль 3), подключи форму + аналитику (модуль 4).
4. Задеплой на поддомен `*.eclipse-forge.ru` (модуль 5).
5. Прогони чеклист — и показывай.

**Готово — ты умеешь делать продающие лендинги и собирать новые за часы.** 🎉
