# 🎓 Курс: интернет-магазин с нуля — учебный формат

> Мини-курс для команды: научиться собирать **рабочие интернет-магазины** быстро и качественно,
> бесплатно на старте, **силами нейросети** (Claude / Codex / Kimi) и наших сервисов.
> Путь от «никогда не делал» до своего headless-магазина на нашем стеке.
> Каждый урок устроен одинаково:
> **🎯 чему научишься · 📖 коротко (теория) · 🛠 практика (с нейросетью) · ✅ чек.**
> Инструменты по этапам → [раздел библиотеки](../README.md#-разработка-интернет-магазинов) · сайт: <https://library.eclipse-forge.ru>

## Программа курса

| Модуль | О чём | Уровень |
|---|---|---|
| **0. Что такое интернет-магазин** | Из чего состоит, анатомия, выбор пути | новичок |
| **1. Первый магазин за вечер** | Сборка нейросетью из готового шаблона | новичок |
| **2. Товары, каталог, контент** | Каталог, карточки, фото и тексты | базовый |
| **3. Корзина, заказ, оплата** | Checkout + эквайринг (ЮKassa) | базовый |

| **5. Запуск** | Хостинг, домен, SEO, аналитика, чеклист | средний |

| **🏁 Финальный проект** | Реальный магазин под клиента | — |

**Как проходить:** по порядку, делая практику в каждом уроке.
**Новичку** хватит модулей 0–5 — магазин соберёт нейросеть по твоим словам, кодить не нужно.
**Разработчику** — весь курс, включая модуль 6 (свой headless-стек с полным контролем).
К концу — рабочий магазин и навык собирать новые за вечер.

> **💡 Главная идея курса:** сегодня магазин собирают **не руками, а словами** — ты объясняешь
> нейросети, что хочешь, она пишет код, ставит зависимости, чинит ошибки и помогает выложить сайт.
> Программировать **не нужно** (до модуля 6) — нужно понятно формулировать и проверять результат.

## Модуль 0. Что такое интернет-магазин

### Урок 0.1 — Анатомия магазина
**🎯 Научишься:** видеть магазин как набор стандартных частей и понимать, зачем каждая.
**📖 Коротко:** магазин в интернете — сайт, где человек может **выбрать товар, положить в корзину и оплатить**. Внутри — простые части:

| Часть | Что это простыми словами |
|---|---|
| **Витрина** | То, что видит покупатель: страницы с товарами, картинки, цены |
| **Каталог** | Список твоих товаров (название, фото, цена, описание) |
| **Корзина** | «Тележка», куда покупатель складывает товары перед оплатой |
| **Оформление заказа (checkout)** | Форма: имя, адрес, телефон, способ оплаты и доставки |
| **Оплата** | Приём денег картой/СБП — через сервис-«платёжку» (эквайринг) |
| **Доставка** | Как товар доедет: почта, СДЭК, самовывоз, курьер |
| **Админка** | Твой «пульт»: добавляешь товары, видишь заказы |

**Хорошая новость:** всё это делать руками не нужно — есть готовые шаблоны и движки, где части уже встроены.
**🛠 Практика:** открой 2–3 знакомых интернет-магазина и найди в них каждую часть из таблицы.
**✅ Чек:** можешь назвать 6+ частей магазина и зачем каждая.

### Урок 0.2 — Выбери свой путь
**🎯 Научишься:** выбирать подход под свою ситуацию и не переусложнять.
**📖 Коротко:** три пути к магазину:

| Путь | Кому | Чем делаем |
|---|---|---|
| 🤖 **С нейросетью** (рекомендуем) | Хочешь свой магазин, но не кодишь | **Claude / Codex / Kimi** пишут код по твоим словам |
| 🟢 **Совсем без кода** | Не хочешь возиться даже с нейросетью | конструктор Tilda / Ecwid (быстро, но шаблонно) |
| 🔴 **Сам пишешь код** | Ты разработчик | Medusa + Next.js → модуль 6 |

Главный путь курса — **собрать магазин нейросетью**: и просто (объясняешь словами), и получаешь настоящий сайт под свой дизайн, в отличие от шаблонного конструктора.
**🛠 Практика:** реши, какой путь твой (по умолчанию — 🤖 с нейросетью).
**✅ Чек:** выбран путь и понятно, почему.

> **Что нужно подготовить:** 1–3 товара (даже для начала), фото (с телефона сойдут — светло и без хлама),
> короткие описания и цены, название магазина, 2–3 часа времени. **Денег на старте не нужно** — начнём бесплатно.

## Модуль 1. Первый магазин за вечер (AI-first)

### Урок 1.1 — Выбери помощника-нейросеть
**🎯 Научишься:** выбирать кодинг-агента и начинать с ним работу.
**📖 Коротко:** магазин соберёт **агент-нейросеть** — он сам создаёт файлы, ставит зависимости, чинит ошибки и деплоит.

| Нейросеть | Чем хороша | Как начать |
|---|---|---|
| **Claude (Claude Code)** | Лучший «агент-программист», наш основной инструмент | команда `claude` в терминале или расширение в VS Code |
| **Codex (OpenAI)** | Агент OpenAI — в ChatGPT и как `codex` CLI | через ChatGPT или Codex CLI |
| **Kimi (Moonshot)** | Kimi Code CLI + браузерный агент (сам кликает) | kimi.com → Kimi Code |

Новичку проще всего: **Claude Code** или **Codex в ChatGPT**. Дальше шаги одинаковые для любой.
**🛠 Практика:** запусти выбранного агента и поздоровайся — попроси «объясни, как мы будем делать магазин по шагам».
**✅ Чек:** агент запущен, отвечает, готов вести по шагам.

### Урок 1.2 — Возьми готовый шаблон, не пиши с нуля
**🎯 Научишься:** стартовать с готового магазина-шаблона вместо «напиши с нуля» (это долго).
**📖 Коротко:** проси взять **готовый бесплатный шаблон** и доработать. Эталон — [Vercel Next.js Commerce](https://github.com/vercel/commerce); если нужен полноценный бэкенд с заказами — шаблон на **Medusa**.
**🛠 Практика:** скопируй агенту:
> Склонируй [Vercel Next.js Commerce](https://github.com/vercel/commerce), установи зависимости и запусти локально. Веди меня по шагам и объясняй простыми словами, что делаешь.

**✅ Чек:** шаблон скачан, зависимости поставлены, проект запускается локально.

### Урок 1.3 — Веди нейросеть словами
**🎯 Научишься:** превращать «что хочу» в результат обычным языком, без кода.
**📖 Коротко:** дальше — готовые фразы (промпты), вставляй и меняй под себя:
> Добавь мои товары: [название, цена, описание для каждого]. Сделай страницу каталога и карточку товара.

> Поменяй шапку: логотип слева, название магазина «…», меню Каталог / О нас / Контакты.

> Сделай корзину и страницу оформления заказа с полями: имя, телефон, адрес, комментарий.

Не понял ответ — пиши «**объясни простыми словами**» или «**покажи, что нажать**». Сломалось — скопируй текст ошибки агенту, он починит.
**🛠 Практика:** дай агенту 2–3 правки словами под свой магазин.
**✅ Чек:** в магазине появились твои товары и нужные пункты меню.

### Урок 1.4 — Запусти и проверь как покупатель
**🎯 Научишься:** смотреть на результат глазами клиента и давать точные правки.
**📖 Коротко:** пройди путь покупателя: товар → корзина → оформить заказ. Понятно ли, удобно ли на телефоне?
**🛠 Практика:**
> Запусти магазин локально и дай мне ссылку — я проверю.

Потыкай как покупатель, что не так — скажи словами («увеличь кнопку», «проверь мобильную вёрстку»).
**✅ Чек:** путь товар→корзина→checkout проходится, на телефоне не разъезжается.

## Модуль 2. Товары, каталог, контент

### Урок 2.1 — Каталог и карточка товара
**🎯 Научишься:** наполнять магазин товарами так, чтобы их было удобно выбирать.
**📖 Коротко:** каталог — список товаров (название, фото, цена, описание); карточка товара — отдельная страница с деталями и кнопкой «в корзину». Группируй товары по категориям, если их много.
**🛠 Практика:**
> Сделай каталог с категориями [список]. На карточке товара: галерея фото, цена, описание, варианты (размер/цвет), кнопка «В корзину».

**✅ Чек:** товары видны в каталоге, карточка открывается, «в корзину» работает.

### Урок 2.2 — Фото, картинки и тексты (наши сервисы)
**🎯 Научишься:** делать контент магазина без дизайнера и копирайтера.
**📖 Коротко:** контент = доверие. Хорошее фото и внятное описание продают лучше низкой цены.
**🛠 Практика (наши сервисы — бесплатно и под рукой):**
- **Фото товаров** → [Shotforge](https://shotforge.eclipse-forge.ru) (AI-фотограф: предметка, стили).
- **Баннеры / иллюстрации** → [Text2Image](https://text2image.eclipse-forge.ru).
- **Описания товаров, SEO-тексты** → [Eclipse AI Hub](https://hub.eclipse-forge.ru) (Copywriter) или просто попроси агента.
- **Парсинг поставщиков/конкурентов** → [Eclipse-Claw](https://github.com/PavelHopson/Eclipse-Claw).

**✅ Чек:** у каждого товара качественное фото и понятное описание.

## Модуль 3. Корзина, заказ, оплата

### Урок 3.1 — Корзина и оформление заказа
**🎯 Научишься:** доводить покупателя от выбора до оформленного заказа без трения.
**📖 Коротко:** корзина хранит выбранное; checkout — форма с минимумом полей (имя, телефон, адрес, способ доставки/оплаты, комментарий). Чем меньше полей — тем больше заказов доходит.
**🛠 Практика:**
> Сделай корзину (плюс/минус количество, удаление) и страницу оформления заказа: имя, телефон, адрес, доставка, оплата, комментарий. После заказа — страница «спасибо».

**✅ Чек:** заказ оформляется, поля минимальны, есть экран подтверждения.

### Урок 3.2 — Как принимать деньги (эквайринг)
**🎯 Научишься:** подключать приём оплаты картой/СБП безопасно.
**📖 Коротко:** чтобы магазин принимал оплату, нужен **эквайринг** — сервис-посредник («платёжка»), который проводит деньги от покупателя к тебе. Сам сайт деньги не «ловит». Новичку в РФ — **[ЮKassa](https://yookassa.ru)** (самый ходовой, готовые SDK/плагины). Альтернативы: [Тинькофф Касса](https://www.tbank.ru/kassa/), [CloudPayments](https://cloudpayments.ru), [Robokassa](https://robokassa.com). Global — [Stripe](https://stripe.com).
**Как подключить (общая схема):**
1. Зарегистрируйся в ЮKassa, заведи «магазин» в их кабинете.
2. Получишь **shopId** и **секретный ключ** — как логин/пароль для приёма денег.
3. Отдай агенту: «подключи ЮKassa, ключи положи в `.env`, не в код».
4. Сделай тестовую оплату — деньги должны пройти.

**🛠 Практика:**
> Подключи оплату ЮKassa. Вот мой shopId и ключ — подскажи, куда их вставить безопасно (в `.env`, не в код). Сделаем тестовый платёж.

> **⚠️ Важно:** чтобы принимать деньги легально, нужен статус (самозанятость/ИП) — это модуль 4.
**✅ Чек:** тестовая оплата проходит, ключи лежат в `.env` (не в коде/репо).

## Модуль 4. Доставка, юр-минимум, обязательные страницы

### Урок 4.1 — Доставка
**🎯 Научишься:** настраивать, как товар доедет до покупателя.
**📖 Коротко:** на старте достаточно **самовывоза** и/или фиксированной стоимости доставки. Дальше — интеграции СДЭК / Почты России / Boxberry (готовые плагины или просьба агенту).
**🛠 Практика:**
> Добавь способы доставки: самовывоз (бесплатно) и доставка по [городу] за [цена]. Покажи выбор на checkout.

**✅ Чек:** на оформлении заказа есть выбор доставки, цена считается.

### Урок 4.2 — Юридический минимум (не страшно)
**🎯 Научишься:** принимать оплату честно и легально.
**📖 Коротко:** чтобы принимать деньги, нужен один из статусов:
- **Самозанятость** — самый простой старт: оформляется за 10 минут в приложении **«Мой налог»** (ФНС), бесплатно, налог 4–6% только с продаж. Подходит, если продаёшь своё/услуги и оборот небольшой.
- **ИП** — если оборот больше или нужны сотрудники/опт. Чуть сложнее, открывается онлайн.

**🛠 Практика:** оформи самозанятость в «Мой налог» (или проверь, что статус уже есть) — без него платёжка не подключит оплату.
**✅ Чек:** есть действующий статус (самозанятость/ИП) под приём оплаты.

### Урок 4.3 — Обязательные страницы
**🎯 Научишься:** добавлять юр-страницы, без которых не подключат оплату.
**📖 Коротко:** обязательный минимум на сайте:
- **Оферта** (договор-условия покупки);
- **Политика конфиденциальности** (как обрабатываешь данные — 152-ФЗ);
- **Контакты** и условия **возврата/доставки**.

Шаблоны этих текстов бесплатно гуглятся за минуту — бери, подставляй свои реквизиты.
**🛠 Практика:**
> Добавь страницы: Контакты, Оферта, Политика конфиденциальности, Возврат — вот мои данные: […]. Поставь ссылки в футер.

**✅ Чек:** все четыре страницы есть и доступны из футера.

## Модуль 5. Запуск

### Урок 5.1 — Хостинг и домен
**🎯 Научишься:** выкладывать магазин в интернет бесплатно.
**📖 Коротко:** витрина → **Vercel** / **Cloudflare Pages** / **Netlify** (бесплатно, превью-деплои) или **наш VPS** через Caddy. Бэкенд (если есть) — наш VPS / Railway / Render. Домен — поддомен `*.eclipse-forge.ru` или свой за ~150 ₽/год.
**🛠 Практика:**
> Задеплой магазин на Vercel (бесплатно) и дай ссылку.

…или на наш сервер:
> Задеплой на наш VPS, домен `магазин.eclipse-forge.ru`. Проведи по шагам.

**✅ Чек:** магазин открыт по рабочей ссылке на HTTPS.

### Урок 5.2 — SEO, Open Graph и аналитика
**🎯 Научишься:** чтобы магазин находили и видели результат.
**📖 Коротко:** title + description, **Open Graph** (превью в соцсетях/мессенджерах), фавикон, sitemap, **schema.org Product** (товары в поиске с ценой). Скорость — проверка в **Lighthouse** (Chrome DevTools), цель 90+, картинки в WebP. Аналитика — **Яндекс.Метрика** (РФ) / **GA4** + цель на оформление заказа.
**🛠 Практика:**
> Добавь meta title/description, OG-теги с превью, schema.org Product на карточки. Прогони Lighthouse и почини красное.

> Подключи Яндекс.Метрику [номер], добавь цель на оформление заказа.

**✅ Чек:** Lighthouse зелёный, ссылка показывает превью, заходы и цель видны в аналитике.

### Урок 5.3 — Чеклист запуска + грабли
**🎯 Научишься:** не забыть важное перед показом клиенту.
**Чеклист «магазин готов к запуску»:**
- [ ] Есть 1+ товар с фото, ценой, описанием
- [ ] Работает корзина и оформление заказа
- [ ] Подключена оплата (тест: успех + отмена + возврат) — или временно «при получении»
- [ ] Настроена доставка (хотя бы самовывоз)
- [ ] Страницы: контакты, оферта, политика, возврат
- [ ] Оформлен статус (самозанятость/ИП)
- [ ] SEO-теги + OG-превью + фавикон, schema.org Product
- [ ] Аналитика и цель подключены
- [ ] Сам сделал тестовый заказ — всё пришло
- [ ] HTTPS, домен, мобильная вёрстка ровная

**Грабли:** оплата «в тест-режиме» забыта в проде · ключи платёжки в коде вместо `.env` · тяжёлые картинки (тормозит) · форма заказа «в никуда» · ломается на мобиле · нет обязательных юр-страниц.
**✅ Чек:** все галочки стоят, тестовый заказ дошёл — можно показывать.

## Модуль 6. Свой код: headless на Medusa (продвинутый)

Для разработчиков, кому нужен полный контроль над UI и стек как у остальных наших проектов (TypeScript end-to-end). Новичкам этот модуль не обязателен — магазин уже собран нейросетью в модулях 1–5.

### Урок 6.1 — Когда нужен свой стек
**🎯 Научишься:** выбирать подход осознанно, а не «всегда headless».
**📖 Коротко:** сначала выбери подход:

| Ситуация | Что брать | Время |
|---|---|---|
| Клиенту «вчера», без кода | **No-code**: Ecwid / Tilda | часы |
| Стандартный магазин, не кодить бэкенд | **Open-source**: WooCommerce (WordPress) | 1–2 дня |
| Кастомный дизайн + контроль, наш стек React/TS | **Headless**: Medusa + Next.js — *наш дефолт* | 2–5 дней |
| Статичный сайт, нужна только корзина | **Cart-as-service**: Snipcart | часы |

**Наш путь — headless на Medusa:** Medusa (Node/TS, REST API) · Next.js App Router + Tailwind (витрина) · Postgres · ЮKassa/Stripe. TypeScript end-to-end, полный контроль над UI, бесплатный self-host.
**🛠 Практика:** выбери подход под задачу (по умолчанию — headless на Medusa для кастомного магазина).
**✅ Чек:** подход выбран и обоснован.

### Урок 6.2 — Подними Medusa + Next.js локально
**🎯 Научишься:** запускать наш стек локально за минуты.
**📖 Коротко:**
```bash
# создаёт backend + admin + (опц.) Next.js storefront в одном репо
npx create-medusa-app@latest my-store
# выбрать: install Next.js storefront = Yes
cd my-store
```
Нужен локальный **Postgres** (или сразу бесплатный [Neon](https://neon.tech) — подставить его `DATABASE_URL`).
```bash
npm run dev            # backend:9000 (admin на /app), storefront:8000
```
- Админка: `http://localhost:9000/app` — товары, цены, регионы, доставка.
- Витрина: `http://localhost:8000` — каталог/корзина/checkout уже работают.

> **⚡ Делай это с агентом:** «подними Medusa + Next storefront», «обвяжи интеграции `retry-http`» — агент пишет код и чинит, ты ревьюишь.
**🛠 Практика:** подними `create-medusa-app`, заведи 2–3 товара в админке, открой витрину.
**✅ Чек:** backend и storefront запущены, товары видны на витрине.

### Урок 6.3 — Каталог, платежи, деплой
**🎯 Научишься:** доводить headless-магазин до прод-готовности.
**📖 Коротко:**
- **Контент/каталог:** товары/категории/варианты — через админку Medusa. Картинки — в S3-совместимое хранилище или [Supabase Storage](https://supabase.com) (free-tier), **не в репо/на диск инстанса** (потеряются при редеплое).
- **Платежи:** РФ → [ЮKassa](https://yookassa.ru) (community-плагин провайдера, `shopId`+`secretKey` в `.env`, включить в регионе RU). Global → [Stripe](https://stripe.com) (официальный модуль, `STRIPE_API_KEY`). Для РФ Stripe напрямую не выйдет — ЮKassa/Тинькофф.
- **Деплой (бесплатный старт):**

| Что | Куда |
|---|---|
| **Витрина (Next.js)** | [Vercel](https://vercel.com) (один клик) или наш VPS через Caddy |
| **Backend (Medusa)** | наш **VPS** (Mara) через [Coolify](https://coolify.io) / systemd · либо [Railway](https://railway.app) / [Render](https://render.com) |
| **Postgres** | [Neon](https://neon.tech) free-tier (или Postgres на VPS) |
| **Файлы/картинки** | Supabase Storage / S3 |
| **Домен** | поддомен `*.eclipse-forge.ru` (DNS A → VPS + Caddy) |

> На нашем VPS проще всего поставить **Coolify** (self-host PaaS) → он поднимет Medusa + Postgres контейнерами и выдаст HTTPS. Витрину — отдельным статик-деплоем через `deploy-vps.yml` или Vercel.

**🛠 Практика:** наполни каталог, подключи ЮKassa в тест-режиме, задеплой витрину на Vercel/VPS, бэкенд — через Coolify.
**✅ Чек:** прод-витрина открывается, тестовая оплата проходит, картинки в внешнем хранилище.

### Урок 6.4 — Грабли headless
**🎯 Научишься:** обходить типичные ошибки на нашем стеке.
**📖 Коротко:**
- **Medusa требует Postgres + Redis** для прода — не забудь оба (Redis: Upstash free-tier).
- **РФ-платежи через Stripe напрямую не выйдут** — бери ЮKassa/Тинькофф/CloudPayments.
- **CORS** между витриной и backend — пропиши домены в `STORE_CORS`/`ADMIN_CORS` Medusa.
- **Картинки товаров** — не храни в репо/на диске инстанса → S3/Supabase.
- **Не пиши витрину с нуля** — бери [Vercel Commerce](https://github.com/vercel/commerce) или Medusa-storefront и допиливай.
- **Запросы к API платёжек/доставки** оборачивай ретраями — наш [`retry-http`](https://github.com/PavelHopson/retry-http).

**Быстрые альтернативы (если headless избыточен):**
- **WooCommerce** — WordPress + плагин + тема Storefront + плагин ЮKassa. Магазин за день.
- **Snipcart** — на любую статичную витрину: `<button class="snipcart-add-item" data-item-...>` даёт корзину и checkout.
- **Tilda** — срочно клиенту: блок «Магазин» + ЮKassa/Робокасса в настройках, без кода.

**🛠 Практика:** пройди свой проект по списку граблей и закрой найденное.
**✅ Чек:** Redis есть, CORS настроен, картинки во внешнем хранилище, ретраи на платёжках.

## ❓ Страхи новичка → честные ответы

**«Я не программист — справлюсь?»** Да. Нейросеть (Claude / Codex / Kimi) пишет код **за тебя** — ты объясняешь словами и проверяешь результат. Совсем не хочешь кода — есть конструктор (Tilda), там всё мышкой.

**«Нужны ли деньги на старте?»** Нет. У нейросетей есть бесплатные/пробные режимы, шаблон магазина бесплатный, хостинг (Vercel / наш VPS) — бесплатно, статус самозанятого — бесплатно. Платишь только когда пойдут продажи (комиссия платёжки ~2–3%).

**«А если нейросеть напишет с ошибкой?»** Скопируй ей текст ошибки — починит. Это нормальный процесс: пробуешь → не так → говоришь словами → правит. Главное — проверять каждый шаг.

**«А если не пойдёт?»** Ничего не теряешь. Начни с малого: 2–3 товара, проверь спрос, потом расширяй.

**«Нужен ли склад/много товара?»** Нет. Можно продавать под заказ, цифровое, услуги, хендмейд.
