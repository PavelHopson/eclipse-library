# 🤖 Cua для device-QA: автоматизируем ручные прогоны (связка с Claude Code)

> Рецепт: заменить ручную проверку «открыл приложение на телефоне → потыкал →
> убедился, что не сломалось» на агента, который делает это сам — управляет
> реальным app по скриншоту (видит кнопки визуально, как человек) в **изолированной
> песочнице**. Источник: [Cua (trycua/cua)](https://github.com/trycua/cua), MIT,
> 19k★ (Подборка Eclipse 23–27.06).
>
> **Куда ложится:** **Eclipse Chat** и **FinFlow** — smoke-прогоны после деплоя
> (логин, ключевой сценарий, «ничего не белоэкранит»), которые сейчас Pavel чекает
> руками. Cua умеет **Android-песочницу** — это прямо про мобильную проверку EC/FinFlow PWA.
>
> ⚠️ **Безопасность (читать первым):** computer-use = агент жмёт кнопки и вводит
> текст за вас. Запускать **только в изолированной среде** (Cua Cloud / локальная
> VM/контейнер), **никогда не на своей рабочей машине с доступом к проду, ключам,
> почте**. Не давать агенту реальные платёжные/админ-креды — только тестовые
> аккаунты. Это не «удобство», а граница безопасности.

## Два пути

**A. MCP-драйвер в Claude Code (рекомендуется для старта).** Cua ставится как
MCP-сервер — и вы гоняете QA-сценарии прямо из Claude Code на естественном языке
(«открой educator.eclipse-forge.ru, залогинься тестовым аккаунтом, собери курс из
каталога, проверь что отрендерились модули»). Минимум кода, агент сам ведёт цикл
скриншот→действие→проверка.

**B. Python SDK (скриптуемые прогоны для CI/повторяемости).** Пишете сценарий
кодом — детерминированный smoke-набор, который можно гонять по расписанию.

## Путь A — Cua как MCP в Claude Code

```bash
# 1. Драйвер (фоновое управление десктопом/песочницей)
#    macOS/Linux:
curl -fsSL https://raw.githubusercontent.com/trycua/cua/main/libs/cua-driver/scripts/install.sh | bash
#    Windows PowerShell:
#    irm https://raw.githubusercontent.com/trycua/cua/main/libs/cua-driver/scripts/install.ps1 | iex

# 2. Подключить как MCP-сервер к Claude Code
claude mcp add --transport stdio cua-driver -- cua-driver mcp
```

Дальше в Claude Code — обычный промпт-сценарий, напр.:

> «В песочнице открой `https://finflow.eclipse-forge.ru`, войди тестовым аккаунтом
> (логин/пароль ниже), создай расход 500 ₽ в категории «Продукты», убедись что
> операция появилась в списке за сегодня. Сделай скриншот результата и скажи,
> прошло ли.»

Агент сам делает скриншоты, находит поля, кликает, и **возвращает вердикт + кадр**.

## Путь B — скриптуемый smoke на Python SDK

```bash
pip install cua        # Python 3.11+
```

```python
# smoke_finflow.py — пример эфемерной песочницы (Linux); для мобилки → Image.android()
from cua import Sandbox, Image

async def main():
    async with Sandbox.ephemeral(Image.linux()) as sb:
        # Открыть приложение в песочнице (через её браузер/CLI),
        await sb.shell.run("chromium https://finflow.eclipse-forge.ru &")
        shot = await sb.screenshot()              # «видим» то же, что человек
        await sb.mouse.click(100, 200)            # клик по координате
        await sb.keyboard.type("test@example.com")
        # … далее: связать скриншот с vision-LLM, чтобы находить элементы по смыслу,
        #    а не по жёстким координатам (см. ниже про устойчивость).

# OS-образы песочницы: Image.linux() / .macos() / .windows() / .android()
```

> API Cua быстро эволюционирует (классы Sandbox/Image, агентный цикл) — перед
> внедрением сверяйтесь с **[cua.ai/docs](https://cua.ai/docs)**. Здесь — проверенный
> на 27.06 минимум (`pip install cua`, `cua-driver mcp`, Android-образ).

## Куда встроить у нас

- **EC / FinFlow post-deploy smoke:** после авто-деплоя (educator/finflow на VPS) —
  агент проходит 1 критический сценарий и шлёт вердикт. Снимает с Pavel ручной чек.
- **Android-песочница = мобильная проверка** PWA EC/FinFlow без физического телефона.
- **Регрессии перед релизом:** набор из 3–5 сценариев (логин, ключевое действие,
  выход) гонять перед мержем в `main`.

## Ограничения / готчи

- **Скорость и стоимость:** запуск VM + vision-цикл медленнее юнит-тестов; это
  **smoke/E2E**, а не замена быстрым тестам. Гонять точечно, не на каждый коммит.
- **Хрупкость по координатам:** жёсткие `click(x, y)` ломаются при смене вёрстки —
  находить элементы через vision-LLM по смыслу («кнопка Сохранить»), как в
  [marketplace-screenshot-parsing.md](marketplace-screenshot-parsing.md).
- **Только тестовые данные/аккаунты** в песочнице. Никаких прод-кредов, ключей,
  платёжек — см. блок безопасности выше.
- **Не для CI на публичных раннерах** без изоляции — computer-control в чужой среде
  опасен; свой изолированный воркер или Cua Cloud.
